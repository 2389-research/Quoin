The biggest release since launch: a wave of editor features, real macOS-native integration, data-safety hardening for synced libraries, a full Swift 6 language-mode migration, and both native engines updated. Every invariant suite (byte-lossless round-trips, the caret/viewport rule, reveal fidelity, projector equivalence, RDFM conformance) stays green.

### Added — editing
- **Text zoom** — ⌘= / ⌘− / ⌃⌘0 scale the reading view; exports and print always render at 100%.
- **Structural commands** (Format ▸ Structure) — set/cycle heading level, toggle bullet/numbered list, toggle block quote, and toggle a checkbox on the caret's line (⌃⌘↩), all as byte-exact source edits.
- **Find & Replace depth** — Match Case, Whole Word, Regex, and In-Selection options, with one shared matcher so find-highlighting and replace always agree.
- **Paste a clipboard image** — ⌘V a screenshot or copied image and it's saved into the library's `assets/` folder with an `![](…)` reference inserted at the caret.
- **List editing** — Tab / ⇧Tab indent and outdent; Return continues bullets, numbers, `- [ ]` checkboxes, and `>` quotes, and ends the list on an empty item.
- **Word-granular undo** — typing coalesces into word-sized undo steps instead of reverting one character at a time.
- **Continuous spell-check** — misspellings underline as you write (autocorrect and smart-substitutions stay off, so your exact bytes reach disk).
- **Wrap Lines** (View menu) — wrap long lines to the column, or let them run with a horizontal scroller.

### Added — macOS integration
- **System Share** — send the current document through the share sheet (AirDrop, Mail, Messages, any extension) from the toolbar.
- **`quoin://open?path=…` deep links** — open a library document by URL, with the path confined to the sandboxed library root.
- **Drag a document (or folder) out** of the sidebar to Finder or another app.
- **Accessibility** — the chrome scales with Dynamic Type, and Reduce Motion and Reduce Transparency are honored.
- **HIG polish** — context-aware Undo/Redo naming ("Undo Typing"), Page Setup (⇧⌘P), Show Next/Previous Tab (⌃⇥ / ⌃⇧⇥), and per-window Focus/Typewriter modes.

### Data safety
- **Coordinated file access for synced libraries** — reads and writes go through `NSFileCoordinator`, and the app registers an `NSFilePresenter`, so an iCloud/Dropbox/Drive library no longer races the sync daemon into "conflicted copy" files; a coordinated read also pulls down an undownloaded iCloud placeholder instead of failing to open it.
- **File-encoding detection** — a `.md` written as UTF-16 or Latin-1 opens correctly and round-trips in its original encoding.

### Fixed
- **Focus Mode** no longer collides with the system Enter-Full-Screen shortcut (it's a View-menu / toolbar action).
- **Outline section highlight** now tracks the section you're actually reading and stays there until the next heading reaches the top (resolved by document order, not viewport-dependent ranges).
- A **file-descriptor leak** in the file watcher that could exhaust the process's descriptors under heavy churn.

### Engines
- **MermaidKit 1.3.0** — entity-relationship diagrams now render PK/FK/UK key badges; existing diagram rendering unchanged.
- **Vinculum 1.4.2** — two math parser data-loss fixes (`\end` no longer consumed after `\bf`/`\color`; double sub/superscripts degrade visibly instead of dropping content).

### Under the hood
- **Swift 6 language mode** for QuoinCore, the macOS app target, and the UI-test bundle, with strict concurrency fixed at the root — no `@preconcurrency` or unchecked escapes. (QuoinRender's caret/viewport machinery stays Swift 5 for now, a documented staged migration.)
- **Large-document render moved off the main actor** (generation-guarded, invariant-preserving) so big files stay responsive.
- Hygiene: hoisted formatter allocations, surfaced previously-silent failures, coalesced library rescans, and added a **Linux CI job**.
